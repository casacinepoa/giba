2. ARQUIVOS E PROGRAMAS

2.1. Arquivos

Arquivos são seqüências de bits, normalmente organizadas em bytes, que podem ser entendidas pelo computador como um conjunto autônomo e funcional de dados. Existem vários tipos de arquivos, conforme o tipo de dados que eles armazenam (embora, no fim das contas, todos eles sejam seqüências de zeros e uns).

Mas a principal classificação dos arquivos é binária: ou eles são arquivos de dados ("data") ou são programas ("software").

2.2. Programas

Programas são os arquivos que armazenam instruções para o computador. Cada conjunto de instruções bem definidas, executáveis, com o objetivo de resolver um problema, pode ser chamado também de algoritmo.

Existem programas para lidar com dados de outros arquivos, para transferir dados de um setor de memória a outro, ou da memória para a tela do computador, ou para a impressora, etc. E existem programas que são conjuntos extremamente complexos de instruções sobre como agir em relação a novas instruções geradas pelo usuário, através do teclado, do mouse, de voz, de reconhecimento facial, etc. Os arquivos de programas também podem ser classificados em: arquivos executáveis, bibliotecas, arquivos do sistema operacional, etc.

2.3. Arquivos de dados

Arquivos de dados são os arquivos que apenas armazenam dados, para que estes sejam manipulados pelos arquivos de programas, de acordo com as instruções do usuário. A partir daqui, vamos estudar 4 tipos principais de arquivos de dados, como eles são gerados e armazenados, e como nós podemos lidar com eles:

- arquivos de texto
- arquivos de imagem
- arquivos de som
- arquivos de imagem em movimento


2.4. Algoritmos

Segundo o Houaiss, algoritmo é uma "sequência finita de regras, raciocínios ou operações que, aplicada a um número finito de dados, permite solucionar classes semelhantes de problemas".

O nome vem de um matemático e astrônomo persa do século IX chamado Muhammad Ibn Musa al-Khwarizmi, o mesmo que também gerou a palavra 'algarismo' (dígito numérico).

Em matemática simples usamos algoritmos para extrair a raiz quadrada de um número, ou o Máximo Divisor Comum de dois números, ou para descobrir se um número é primo, etc. Engenheiros gostam de dizer que nós usamos algoritmos no dia-a-dia, para decidir onde vamos almoçar, que filmes vamos ver ou até por quem vamos nos apaixonar (!).

Algoritmos podem se tornar complexos demais pela quantidade de regras que envolvem a tomada de decisões, ou pelo número de vezes que ele exige repetir determinados procedimentos. Nesse caso, uma máquina (um computador) pode resolvê-lo bem mais rápido e, em princípio, sem erro.

Um algoritmo pode ser apresentado em forma de fluxograma, ou ser escrito diretamente em linguagem de máquina: nesse caso ele é um programa de computador (software). Mas também pode ser escrito como uma série de instruções executáveis por um ser humano.

2.5. Exemplo de algoritmo
(para divisão de números inteiros com mais de dois dígitos)

(1) Colocar o dividendo em A e o divisor em B.

(2) Copiar A para C.

(3) Retirar o dígito mais à esquerda de C
    e colocá-lo à direita de D.

(4) Comparar D com B:
    se D < B, voltar para (3).

(5) Fazer a divisão inteira de D/B:
    colocar o resultado inteiro à esquerda de E;
    colocar o resto em F (sobrescrever).

(6) Conferir se ainda tem dígitos em C:
    se não tem, pular pra (9).

(7) Copiar o valor de F para D (sobrescrever).

(8) Voltar para (3).

(9) A divisão está feita.
    A / B = E e resta F
