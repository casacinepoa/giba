3. ARQUIVOS DE TEXTO

ASCII - (American Standard Code for Information Interchange) Código usado para armazenar informação correspondente a um caractere (letra, número ou símbolo) no espaço de memória de 1 byte (8 bits). Pode codificar, portanto, 256 caracteres diferentes.

tabela de valores em ASCII

Na verdade, o código ASCII só estabelece valores fixos para metade das possibilidades de um byte, ou apenas 128 diferentes caracteres (ASCII 0 a 127): 26 letras maiúsculas, 26 minúsculas, 10 algarismos, 33 sinais (pontuação, acentuação, etc.) e 33 códigos de controle.

As outras 128 combinações são usadas para caracteres especiais, como letras acentuadas, letras de outros alfabetos, etc. Estes caracteres (ASCII 128 a 255) podem assumir valores diferentes em diferentes codificações (EBCDIC, ANSI, OEM, ISO, Unicode, etc.), causando algumas confusões quando o computador tenta ler o texto usando o sistema errado.

diferentes tabelas de ASCII estendido:
ANSI (American National Standards Institute)
OEM (Original Equipment Manufacturer)
ISO-8859-1 (International Standardisation Organization)

Unicode - padrão criado em 1991 para representar dezenas de milhares de caracteres diferentes, usando dois bytes (16 bits) para cada caractere. Seus primeiros 128 carcateres são exatamente iguais aos da "primeira página" ASCII.

UTF-8 (Unicode Transformation Format, versão 8) - variação do Unicode com comprimento variável: os caracteres básicos são representados por 1 byte; os acentuados, por 2 bytes.

TXT - Arquivo de texto em que são armazenados apenas seqüências de caracteres (ASCII ou equivalente), sem formatação.

Um arquivo TXT com 1000 caracteres, codificado em ASCII e suas variações, ocupa exatamente 1000 B (1 kB) de memória - não esquecendo que cada espaço é um caractere e cada quebra de linha pode ser 1 ou 2 caracteres.

Se este mesmo arquivo for codificado em Unicode, vai ocupar 2000 bytes. Em UTF-8, vai ficar entre 1000 e 2000 bytes, dependendo do número de caracteres acentuados.

Outros tipos de arquivo de texto, como DOC (do Word), ODT (do Write) ou RTF ("Rich Text Format"), armazenam muitas outras informações além do texto (formatação, fonte, tabelas, figuras, alterações, dados do usuário, etc.) e por isso ocupam mais espaço.
